# Case study template — social systems

Every folder in `social-systems/` (other than `cross-system/` and this
template) has this layout:

| File | Purpose |
|---|---|
| `overview.md` | One-page summary: what the system is, where it occurs, why it is interesting. |
| `is-it-a-system.md` | Answers the question from research step **6a**: does this match our system definition? Boundary, components, environment, emergent behaviour. |
| `se-decomposition.md` | Answers **6b**: the full five-level SE decomposition (Goals, Requirements, Functions, Logical Architecture, Physical Implementation) as a listing. This is the canonical copy — the book may reference or excerpt it, but must not duplicate it. |
| `historical-se-interventions.md` | Answers **6c**: documented cases where SE-like reasoning was applied historically, without anyone calling it SE. |
| `applied-se-analysis.md` | Answers **6d**: new application of SE techniques (STPA, variation-point analysis, etc.) to this system. |
| `sources.md` | Citations for everything above. |

Copy this folder to a new name under `social-systems/` and fill in
each file. Delete headings that do not apply, but keep the file names
so tooling can walk the catalogue mechanically.
