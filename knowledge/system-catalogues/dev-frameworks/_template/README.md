# Case study template — dev frameworks

Every folder in `dev-frameworks/` (other than `cross-framework/` and
this template) has this layout:

| File | Purpose |
|---|---|
| `overview.md` | One-page summary: what the framework prescribes, where it originated, where it is used. |
| `se-decomposition.md` | The canonical five-level decomposition treated as if the framework were a product to be built. |
| `variation-point-bindings.md` | How the framework binds the universal variation points (see `cross-framework/variation-points.md`). |
| `sources.md` | Citations. |

Compared to the social-systems template, this one omits
`is-it-a-system.md`, `historical-se-interventions.md`, and
`applied-se-analysis.md` — frameworks are themselves the SE-flavoured
intervention, so those questions do not apply in the same form.
