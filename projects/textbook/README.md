# Textbook (placeholder)

A future derived textbook drawing on `../../knowledge/`. Unlike the
`systems-introduction-book/` — which is a narrative introduction for
readers new to SE — this project is intended as a textbook for a
taught course: structured chapters, exercises, and worked examples.

## Status

**Placeholder only.** Nothing has been written. This folder exists
so that the directory structure documented in `../README.md` is
complete.

## Intended scope

- **Audience.** Students on a systems-thinking or institutional-
  design course, with some prior exposure to either engineering or
  social science.
- **Shape.** Chapters map to the four `knowledge/se-techniques/`
  areas (SE hierarchy, product-line engineering, STAMP/STPA,
  control structures and remedies), each followed by a chapter
  applying the technique to a system from
  `knowledge/system-catalogues/`.
- **Exercises.** Each chapter ends with structured exercises that
  ask the student to decompose a new system, find a variation
  point, sketch a control structure, or propose a remedy — with
  reference solutions hidden until the student commits.

## Rule of thumb

As with the other projects, the textbook may reference, excerpt,
and summarise `knowledge/` but must not become a parallel canonical
store of decompositions or techniques.
