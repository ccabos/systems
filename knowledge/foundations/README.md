# Foundations

What a system *is*, before any engineering technique gets applied to it.

## Files

- `system-definition.md` — the working definition used across the project
- `boundaries-and-environment.md` — scope, boundary, environment
- `emergence-and-behavior.md` — emergent properties, behaviour vs structure
- `technical-vs-non-technical.md` — what distinguishes a non-technical system

Each file is a knowledge page: claim, evidence, citations. Narrative
retellings for non-specialist readers live in the book, not here.
