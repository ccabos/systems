# System Catalogues

Case studies of real-world systems, grouped by kind. Each catalogue has
its own `_template/` folder that shows the expected file layout for a
new case study, plus one folder per catalogued system.

## Current catalogues

| Catalogue | What it catalogues |
|---|---|
| `social-systems/` | Religions, polities, voluntary associations, corporations, universities, families, etc. — the ten-system set that Part II of the book explores. |
| `dev-frameworks/` | Software and product development process frameworks: Waterfall, V-Model, PRINCE2, Scrum, Kanban, Design Thinking, DevOps, SAFe — the eight frameworks that Part III of the book explores. |

## Expected future catalogues

As the project scope grows, add catalogues under this folder for other
families of systems, for example:

- `infrastructure-systems/` — power grids, water, transport networks
- `legal-systems/` — civil law, common law, customary law, international law
- `biological-systems/` — ecosystems, immune system, homeostasis
- `financial-systems/` — central banks, markets, payment rails

Each new catalogue must provide a `_template/` folder with the skeleton
files its case studies will use, and a short README explaining what it
catalogues and how it selects members.
