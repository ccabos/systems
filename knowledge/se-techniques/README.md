# SE Techniques

Catalogue of Systems Engineering skills that the project draws upon when
analysing systems. Each technique has its own folder with:

- `overview.md` — one-paragraph definition and purpose
- `how-it-works.md` — the mechanics of applying it
- `technical-examples/` — worked examples in traditional engineering
  domains, where the technique was developed
- `references.md` — sources

## Techniques catalogued

| Technique | Folder |
|---|---|
| Goals / Requirements / Functions / Logical / Physical hierarchy | `goals-requirements-hierarchy/` |
| Product Line Engineering — platform, variation points, reuse | `product-line-engineering/` |
| STAMP — System-Theoretic Accident Model and Processes | `stamp/` |
| STPA — System-Theoretic Process Analysis | `stpa/` |
| Control structures | `control-structures/` |
| Justificatory rungs (extension to STPA for social systems) | `justification-rungs/` |
| Linear algebra — spectral and factorisation methods on existing matrix-shaped artefacts | `linear-algebra/` |

More will be added as scope grows (e.g. FMEA, HAZOP, FTA, model-based
systems engineering).
