# Design Thinking — SE Decomposition

Extracted from `projects/systems-introduction-book/docs/part3/frameworks.md`
§2.6. IDs use the `DT*` prefix as in the source.

## Goals (DTG)

- **DTG1** — Solve problems that truly matter to real people (human-desirability first)
- **DTG2** — Generate innovative solutions through structured creativity
- **DTG3** — Reduce risk of building the wrong thing by validating early and often

## Requirements (DTR)

- **DTR1** — Deep empathy with end users before defining the problem
- **DTR2** — Divergent-then-convergent thinking at each stage
- **DTR3** — Rapid, low-fidelity prototyping to make ideas tangible
- **DTR4** — Testing with real users, not just internal review
- **DTR5** — Cross-disciplinary collaboration (design, engineering, business, users)
- **DTR6** — Iteration — loop back freely between stages based on learning

## Functions (DTF)

- **DTF1** — Empathize — observe, interview, immerse in user context
- **DTF2** — Define — synthesize insights into an actionable problem statement (HMW, POV)
- **DTF3** — Ideate — brainstorm widely, then converge on promising concepts
- **DTF4** — Prototype — build quick, cheap, tangible representations
- **DTF5** — Test — put prototypes in front of real users, observe, learn
- **DTF6** — Iterate — loop back to any earlier stage based on test results

## Logical Architecture (DTL)

- **DTL1** — Research Subsystem — user research, field observations, personas
- **DTL2** — Synthesis Subsystem — affinity mapping, problem framing, HMW questions
- **DTL3** — Ideation Subsystem — brainstorming, concept sketches, selection
- **DTL4** — Prototyping & Test Subsystem — prototyping tools, user testing sessions

## Physical Implementation (DTP)

- **DTP1** — Interview guides, observation notes, empathy maps, persona boards
- **DTP2** — Affinity wall (sticky notes), POV statements, HMW questions
- **DTP3** — Sketches, paper prototypes, Figma/Balsamiq mockups, physical models
- **DTP4** — User test sessions, feedback capture grids, iteration logs
